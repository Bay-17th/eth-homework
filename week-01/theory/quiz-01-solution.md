**답변:**
[여기에 답변을 작성하세요]
블록체인의 핵심 목적은 “신뢰할 수 있는 제3자 없이도 다수가 동일한 데이터(장부)에 대해 합의하고, 그 데이터를 위변조로부터 보호”하는 것이다. 중앙화된 데이터베이스는 모델 상 특정 제3자를 무조건적 신뢰해야하며, 그 제3자가 위변조 혹은 마음대로 내용을 읽는 것을 막을 수 없다. 블록체인은 이에 반해 신뢰 최소화 모델을 목표로 한다.

## 문제 3: Solidity 기초

다음 Solidity 코드에서 `public`과 `view` 키워드의 의미를 각각 설명하세요.

```solidity
function count() public view returns (uint256) {
    return _count;
}
```

**답변:**
- `public`: 이 함수는 컨트랙트 외부(지갑, 다른 컨트랙트, 프론트엔드 등)에서도 호출 가능하도록 노출시킨다는 의미
- `view`: 이 함수는 블록체인의 상태를 읽기만 하고, 절대 변경하지 않는다는 것을 컴파일러와 EVM에게 알려줌

## 문제 4: 상태 변수

Solidity에서 상태 변수(state variable)와 지역 변수(local variable)의 차이점을 설명하세요.

**답변:**
[여기에 답변을 작성하세요]
상태변수는 영구저장되는 storage에 있으며 읽기/쓰기 비용이 지역변수에 비해 비싸고 트랜잭션 종료 후에도 존재한다. 지역변수는 임시 변수용 공간인 memory에 존재하며 읽기/쓰기 비용이 미미한 편이고 트랜잭션 종료시 소멸한다.
