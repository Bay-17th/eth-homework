## 문제 1: 블록체인 기초

블록체인의 핵심 목적은 신뢰할 수 있는 제3자 없이도 여러 참여자가 동일한 데이터에 합의하고 그 데이터가 사후에 임의로 위변조되기 어렵게 유지되도록 하는 것입니다.

중앙화 DB는 한 운영자가 read, write 권한을 가지며 최종 진실을 결정합니다. 
반면 블록체인은 합의 규칙과 검증을 통해 누구나 동일한 규칙으로 상태를 재현하고 특정 한 주체가 임의로 기록을 바꾸기 어렵습니다.

## 문제 2: 이더리움의 특징

비트코인은 주로 가치 이전에 초점을 둔 제한적인 스크립트를 제공하고 이더리움은 EVM 위에서 완전한 스마트 컨트랙트를 실행할 수 있도록 설계되어 있습니다.

이더리움에서는 단순 송금뿐 아니라 토큰, NFT, DEX 같은 프로그램 가능한 상태 전이를 체인 위에서 구현할 수 있습니다.

## 문제 3: Solidity 기초

```solidity
function count() public view returns (uint256) {
    return _count;
}
```

- `public`: 컨트랙트 내,외부에서 모두 호출 가능.
- `view`: 상태를 변경하지 않음을 의미합니다.
          호출자가 eth_call로 조회할 수 있고 트랜잭션으로 실행하더라도 이 함수 자체는 상태를 바꾸지 않습니다.

## 문제 4: 상태 변수

상태 변수는 컨트랙트의 스토리지에 저장되어 트랜잭션이 끝나도 값이 유지됩니다.
읽기, 쓰기는 영구 상태를 건드리므로 비용이 큽니다.

지역 변수는 함수 실행 중에만 존재하며 보통 memory에 놓입니다.
함수 실행이 끝나면 사라지고 상태를 직접 저장하지 않으므로 상대적으로 비용이 작습니다.

## 문제 5: Gas 개념

Gas는 EVM에 붙는 작업량의 단위 이자 비용 모델입니다

Gas가 필요한 이유는 무한 루프 같은 과도한 연산으로 네트워크를 마비시키는 것을 방지하고, 블록에 담을 수 있는 연산량을 제한해 네트워크 자원을 공정하게 배분합니다. 또 검증자가 연산 등에 대한 대가를 받도록 하기 위함입니다
