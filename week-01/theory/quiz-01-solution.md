## 문제 1: 블록체인 기초

블록체인의 핵심 목적은 무엇인가요? 중앙화된 데이터베이스와 비교하여 설명해주세요.

**답변:**
블록체인의 핵심 목적은 모든 노드가 데이터를 분산 저장하고 서로 검증함으로써 시스템 유지에 필요한 신뢰 비용을 최소화하는 데 있다. 중앙화된 데이터베이스는 중앙 관리자를 전적으로 신뢰해야 하는 반면, 블록체인은 정해진 코드와 규칙 그 자체를 신뢰 기반으로 삼기 때문에 신뢰 비용 측면에서 훨씬 합리적이고 효율적인 시스템이다.

## 문제 2: 이더리움의 특징

이더리움이 비트코인과 다른 점은 무엇인가요? 스마트 컨트랙트의 관점에서 설명해주세요.

**답변:**
이더리움은 스마트 컨트랙트를 실행하여 블록체인의 상태를 자유롭게 변화시키는 프로그래밍 가능한 블록체인이다. 비트코인은 단순히 화폐의 거래 내역만 기록하는 반면, 이더리움은 스마트 컨트랙트에 따라 다양한 연산을 수행하고 그 결과를 상태에 반영할 수 있다는 점에서 차이가 있다.

## 문제 3: Solidity 기초

다음 Solidity 코드에서 `public`과 `view` 키워드의 의미를 각각 설명하세요.

```solidity
function count() public view returns (uint256) {
    return _count;
}
```

**답변:**
- `public`: 누구나 호출 가능
- `view`: 수정은 불가, 데이터를 읽기만 함->가스비가 안 듦

## 문제 4: 상태 변수

Solidity에서 상태 변수(state variable)와 지역 변수(local variable)의 차이점을 설명하세요.

**답변:**
상태변수는 컨트랙트 저장소에 영구적으로 저장되어 블록체인에 기록되는 변수이며, 함수 외부(보통 최상위)에 선언된다. 반면 지역변수는 함수 내부에서 선언되어 해당 함수 실행 시에만 메모리에 존재하고, 함수 종료 시 소멸된다.

## 문제 5: Gas 개념

이더리움에서 Gas란 무엇이며, 왜 필요한가요?

**답변:**
Gas란 EVM상에서 트랜잭션을 실행하거나 스마트 컨트랙트를 작동시킬 때 소모되는 연산 자원의 측정 단위이다. 필요한 이유는 악의적인 사용자가 무한 루프나 스팸 트랜잭션을 발생시켜 네트워크를 마비시키는 DDoS 공격을 방지하기 위함이다. 또한 네트워크의 연산 자원을 사용하는 대가로 수수료를 지불하게 하여, 블록체인을 유지하는 검증자들에게 경제적 유인을 제공하는 역할도 한다.
