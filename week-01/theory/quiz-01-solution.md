## 문제 1: 블록체인 기초

블록체인의 핵심 목적은 무엇인가요? 중앙화된 데이터베이스와 비교하여 설명해주세요.

**답변:**
블록체인의 본질적인 목적은 단순한 분산 저장이 아니라, 권한의 집중을 제거하고 네트워크 구조 자체의 검증가능성과 투명성을 기반으로 제 3자 없이도 거래나 기록의 일관된 상태를 유지하는데 있다고 생각합니다. 

중앙화된 데이터베이스는 특정 주체가 데이터를 통제하므로 사용자가 그 주체를 신뢰해야하지만, 블록체인은 분산된 참여자들의 합의 알고리즘과 암호학을 통해 데이터를 같이 검증함으로써 신뢰구조를 알고리즘으로 대체한다는점에서 다르다고 생각합니다.

## 문제 2: 이더리움의 특징

이더리움이 비트코인과 다른 점은 무엇인가요? 스마트 컨트랙트의 관점에서 설명해주세요.

**답변:**
비트코인과 이더리움 모두 스마트 컨트랙트 기능을 가지고 있지만, 설계 철학과 범위에서 근본적인 차이가 있다고 생각합니다. 비트코인 스마트컨트랙트는 보안성과 단순성을 최우선으로 하기때문에 제한된 기능만 사용하고, 주로 거래 조건 일회성 검증과 같은 단순한 로직에만 사용되빈다. 반면, 이더리움은 가상머신 위에서 실행되어 복잡한 조건, 상태 저장, 반복 구조등을 포함하는 일반적인 프로그램 수준의 로진을 구현할 수 있어서 NFT마켓플레이스, 게임등 dApp의 기반으로 사용됩니다.

## 문제 3: Solidity 기초

다음 Solidity 코드에서 `public`과 `view` 키워드의 의미를 각각 설명하세요.

```solidity
function count() public view returns (uint256) {
    return _count;
}
```

**답변:**
- `public`:지갑을 가진 사람, 다른 스마트 컨트랙트, 자기자신등 누구든지 제한없이 호출할 수 있음을 의미하며 자바의 public메서드와 비슷합니다.
- `view`: C++의 const멤버함수나 자바의 단순 Getter 처럼 데이터를 수정하지 않고, 오직 읽기만 하겠다는 뜻입니다. 즉, 가스비가 들지 않는 조회 전용 제한자입니다.

## 문제 4: 상태 변수

Solidity에서 상태 변수(state variable)와 지역 변수(local variable)의 차이점을 설명하세요.

**답변:**
Solidity 에서 상태변수는 프로그래밍 언어에서의 전역 변수와 유사하지만, 단순한 전역메모리가 아니라 블록체인 저장소에 영구적으로 기록된다는 점에서 다릅니다.스코프가 컨트랙트 전체이며 변수의 생존기간은 컨트랙트가 블록체인에 존재하는 동안 지속되고, 값이 변경되면 상태전이가 발생합니다. 반면 지역변수는 함수 내부에 선언되는 블록 스코프 변수이고, 스택이나 메모리에 일시적으로 할당되기 때문에 함수 실행이 끝나면 블록체인 상태에 아무런 영향을 주지 않고 소멸합니다.

## 문제 5: Gas 개념

이더리움에서 Gas란 무엇이며, 왜 필요한가요?

**답변:**
이더리움의 Gas 는 블록체인 위에서 코드를 실행할 때 드는 연산비용입니다. 이더리움은 누구나 코드를 실행할 수 있는 분산 컴퓨팅 환경이기 때문에 누군가가 무한 반복이나 과도한 연산으로 네트워크를 마비시키는 문제를 방지하기 위한 장치가 요구됩니다. Gas는 각 연산에 비용을 부과함으로써 사용자가 계산 자원에 대한 대가를 지불하도록 하고 채굴자에게 연산 수행에 대한 보상을 제공하여 네트워크 안정성을 보장하기 때문에 꼭 필요합니다.
