## 문제 1: 블록체인 기초

블록체인의 핵심 목적은 무엇인가요? 중앙화된 데이터베이스와 비교하여 설명해주세요.

**답변:**
[블록체인의 핵심 목적은 중앙화된 신뢰 기관 없이도 데이터의 무결성과 투명성을 유지하는 것이다. 중앙화된 데이터베이스는 관리자가 데이터를 임의로 수정하거나 삭제할 수 있고 관리 서버가 공격받으면 전체 시스템이 마비될 위험이 있지만, 블록체인은 모든 참여자(노드)가 동일한 데이터를 분산하여 저장하므로 데이터 위변조가 불가능에 가깝고 탈중앙화된 신뢰를 제공한다.]

## 문제 2: 이더리움의 특징

이더리움이 비트코인과 다른 점은 무엇인가요? 스마트 컨트랙트의 관점에서 설명해주세요.

**답변:**
[비트코인이 주로 가치 전송 수단으로서의 기능을 수행한다면, 이더리움은 그 위에 스마트 컨트랙트를 실행할 수 있는 기능을 더해 전 세계의 컴퓨터 역할을 한다. 비트코인의 스크립트 언어는 기능이 제한적이지만, 이더리움은 Solidity를 사용하여 복잡한 조건문과 반복문이 포함된 다양한 탈중앙화 애플리케이션(dApp)을 구현할 수 있다.]

## 문제 3: Solidity 기초

다음 Solidity 코드에서 `public`과 `view` 키워드의 의미를 각각 설명하세요.

```solidity
function count() public view returns (uint256) {
    return _count;
}
```

**답변:**
- `public`: 함수의 접근 범위를 지정하는 키워드로, 컨트랙트 내부뿐만 아니라 외부에서도 이 함수를 호출할 수 있음을 의미한다.
- `view`: 함수의 성격을 규정하는 키워드로, 블록체인에 저장된 상태값을 읽기만 하고 수정하지 않는다는 것을 명시한다. 조회 목적일 때, 이 함수는 실행 시 가스비를 소모하지 않는다.

## 문제 4: 상태 변수

Solidity에서 상태 변수(state variable)와 지역 변수(local variable)의 차이점을 설명하세요.

**답변:**
[상태 변수는 컨트랙트 내부에서 선언되어 블록체인에 영구적으로 저장되는 변수로, 트랜잭션이 끝나도 값이 유지되고, 변경 시 많은 가스비가 발생한다. 지역 변수는 함수 내부에서 선언되어 함수가 실행되는 동안에만 메모리에 임시로 저장되는 변수이다. 함수 종료 시 사라지고, 가스비가 저렴하다.]

## 문제 5: Gas 개념

이더리움에서 Gas란 무엇이며, 왜 필요한가요?

**답변:**
[가스는 이더리움 네트워크에서 트랜잭션을 실행하거나 스마트 컨트랙트를 처리할 때 지불하는 수수료의 단위이다. 이는 네트워크 자원의 무분별한 사용을 막고, 무한 루프와 같은 악의적인 공격으로부터 네트워크를 보호하기 위해 반드시 필요하다. 사용자는 실행하려는 작업의 복잡도에 따라 정해진 양의 가스를 지불해야 한다.]
